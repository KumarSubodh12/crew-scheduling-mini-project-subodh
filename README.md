# ADA Lab Experiment 4

## Backtracking, Branch and Bound & String Matching Mini Project

---

### 📌 Project Overview

This project demonstrates advanced algorithmic techniques used to solve complex real-world problems involving constraints and large search spaces. It includes Backtracking, Branch and Bound, and String Matching algorithms with performance analysis.

---

### 🎯 Objectives

* Implement Backtracking for constraint satisfaction problems
* Apply Branch and Bound to optimize search space
* Implement String Matching algorithms
* Analyze algorithm performance using execution time
* Understand scalability challenges in NP-hard problems

---

### ⚙️ Algorithms Implemented

#### 🔹 Backtracking (Crew Scheduling)

* Assign flights to crew members
* Ensures no overlapping schedules
* Explores all possible combinations

---

#### 🔹 Branch and Bound

* Optimized version of backtracking
* Prunes infeasible paths
* Improves performance

---

#### 🔹 String Matching

* Naive String Matching Algorithm
* Finds pattern occurrences in text
* Counts character comparisons

---

### 📊 Performance Analysis

* Execution time measured using Python `time` module
* Observed increase in complexity with input size
* Demonstrates inefficiency of brute-force approaches

---

### ▶️ How to Run

1. Open the notebook in Google Colab
2. Run all cells sequentially
3. Observe outputs and analysis

---

### 📁 Repository Structure

* `crew_scheduling_notebook.ipynb` → Main implementation

---

### 📌 Key Observations

* Backtracking is powerful but computationally expensive
* Branch and Bound reduces unnecessary computations
* String matching complexity increases with text size
* NP-hard problems do not scale efficiently

---

### 📌 Real-World Applications

* Airline crew scheduling
* Timetable generation
* Search engines (pattern matching)
* Cybersecurity (pattern detection)

---

### 📌 Conclusion

* Backtracking ensures correctness but is slow
* Branch and Bound improves efficiency using pruning
* String matching is widely used in real-world systems
* Efficient algorithms are required for large-scale problems
